## 1. 프로덕트 비전

**핵심 가치**: "자식들이 이 앱만 깔아주면 안심하고 지낼 수 있도록"

시니어 챗(Senior Chat)은 디지털 소외 계층인 시니어를 위한 AI 동반자 앱이다.
따뜻한 손자/손녀 같은 AI가 어르신의 모든 궁금증과 문제를 해결해주는 만능 안심 도우미.

### 투 사이드 제품

| 역할 | 대상 | 핵심 가치 |
|------|------|-----------|
| 사용자 | 60세 이상 시니어 | 무엇이든 물어볼 수 있는 따뜻한 AI 동반자 |
| 구매자 | 자녀/손자녀 | 부모님에게 깔아드리면 안심할 수 있는 앱 |

---

## 2. 홈 화면

2x2 그리드에 4가지 핵심 기능만 표시. 다른 요소 없이 명확하게.

| 위치 | 기능 |
|------|------|
| 좌상 | AI 도우미 |
| 우상 | AI 말벗 |
| 좌하 | 일정 알리미 |
| 우하 | 시간 보내기 |

---

## 3. AI 도우미 (Helper)

스마트 렌즈를 활용해서 어려운 서류를 요약하고, 실생활의 모르는 물건이 뭔지 알려준다.

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 미친듯이 간결한 디자인 | 앱 열자마자 하단 중앙에 커다란 [사진 찍어 물어보기] 버튼 하나만 |
| 느린 말투 | 빠르면 못 읽으니까 결과 텍스트를 천천히 읽을 수 있게 |
| 결론 중심 | "2024년 세제 개편안에 따른..." (X) -> "세금 5천 원 내라는 종이예요" (O) |

### 사용 예시

| 대상 | AI 요약 |
|------|---------|
| 약 봉투/약병 | "이건 아침 먹고 바로 드시는 혈압약이에요." / "주의! 이 약은 우유랑 같이 드시면 안 돼요" |
| 고지서/안내문 | "나라에서 10만 원을 준다는 편지예요." |
| 모르는 물건 | "이건 블루투스 스피커예요. 전원 버튼은 옆에 있어요." |

### API

- `POST /api/v1/helper/analyze` (base64 이미지 + 선택적 질문)

## 4. AI 말벗 (Companion)

외로움과 심심함 해결. **전화를 거는 컨셉**.

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 푸시알림 | "전화왔어요", "말동무 필요하세요?" - 어플을 먼저 찾기 전에 유저를 찾아줌 |
| 과거를 먹고 사는 대화 | "지난번에 말씀하신 그 빨간 꽃, 베란다에서 잘 피었나요?" - 전 대화를 기억해서 꺼냄 -> AI를 '내 얘기를 귀담아듣는 진짜 친구'로 느끼게 |
| 처음에 이름 지어주기 | 전화를 거는 컨셉이므로 AI 말벗의 이름을 유저가 지어줄 수 있음 |
| 전화 스타일 UI | 전화화면 같은 디자인, but 간결 |
| 나이대/사투리 설정 | 20대~70대 자유 선택 + 지역 사투리(표준어/경상도/전라도/충청도/제주도) |

### 설정 항목

- 말벗 이름 (기본: 다솜이)
- 말벗 나이 (20~70대)
- 말투/사투리 (표준어, 경상도, 전라도, 충청도, 제주도)

### API

- `GET/PUT /api/v1/companion/{userId}/profile`
- `POST /api/v1/companion/chat` (conversationId로 대화 맥락 유지)

## 5. 일정 알리미 (Schedule)

### UI 구성

- 홈에서 선택하면 바로 **달력** 띄우기
- 날짜 클릭하면 그날 일정 보여주기
- 달력 밑에 **'일정 추가' 버튼**만 크게

### 일정 등록 - 한 화면에 딱 하나만

| 단계 | 질문 | 입력 방식 |
|------|------|-----------|
| 1단계 | "무엇을 하시나요?" | 직접 입력 |
| 2단계 | "언제인가요?" | 날짜 선택 |
| 3단계 | "몇 시인가요?" | 시간 선택 |
| 4단계 | "일정 시작 얼마 전에 연락 드릴까요?" | 알림 시간 선택 |
| 완료 | "일정 등록이 끝났어요!" | 등록 내용 확인 |

### API

- `GET /api/v1/schedules/{userId}`
- `POST /api/v1/schedules`

## 6. 시간 보내기 (Pastime)

### 두뇌/인지 게임 (치매 예방)

| 게임 | 설명 |
|------|------|
| 카드 뒤집기 | 같은 과일/꽃 그림 카드 2장을 연속으로 찾기 (8쌍 16카드) |
| 순서 맞추기 | 1~16까지 숫자를 순서대로 탭 |
| 색깔 글자 맞추기 | 글자의 색깔을 맞추기 (스트룹 효과) |

### 오늘의 퀴즈

- 매일 아침 8시, 저녁 8시에 모든 유저에게 동일한 **상식퀴즈 8문제** 보내기 (푸시알림)
- "지금 000명의 사람들이 함께 풀고 있어요" 표시
- 맞춘 개수에 따른 랭킹:

| 맞힌 수 | 순위 |
|---------|------|
| 8개 | 상위 1% ~ 15% |
| 7개 | 상위 16% ~ 30% |
| 6개 | 상위 31% ~ 50% |
| 5개 | 상위 51% ~ 70% |
| 4개 이하 | 상위 71% ~ 99% |

### 그냥 게임

- 끝말잇기 (AI와 단어 대결)
- 추후 더 추가 가능

### API

- `GET /api/v1/pastime/daily-quiz/questions`
- `POST /api/v1/pastime/daily-quiz/submit`
- `POST /api/v1/pastime/word-chain`

## 7. 시니어 UX 원칙

| 항목 | 상세 |
|------|------|
| 대비 | 모든 화면에서 배경과 글씨의 대비가 명확해야 함 |
| 홈 화면 | 2x2 그리드 대형 버튼 (FeatureCard), 4가지만 표시 |
| 글씨 | 기본 20px 이상, "크게/보통/아주크게" 선택 |
| 버튼 | 56px 이상, 아이콘 + 텍스트 항상 병행 |
| 색상 | 따뜻한 톤, 대비 4.5:1 이상 |
| 피드백 | 모든 액션에 명확한 시각적/청각적 피드백 |
| 네비게이션 | 항상 뒤로가기 가능, 깊은 중첩 없음 |

## 8. 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 모바일 | React Native / Expo 52 | 크로스플랫폼, 앱스토어 배포 |
| 백엔드 | Spring Boot (Java/Kotlin) | 엔터프라이즈급, 검증된 생태계 |
| AI | Spring AI + Anthropic Claude | 서버사이드 프롬프트 관리, API 키 보호 |
| DB | PostgreSQL | 대화 이력, 사용자, 구독 |
| 캐시 | Redis | 세션, 대화 컨텍스트 임시 저장 |
| STT | Whisper API (or Google Speech) | 한국어 음성 인식 |
| TTS | OpenAI TTS (or Clova) | 자연스러운 한국어 음성 합성 |
