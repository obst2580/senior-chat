# Seniar Chat - Product Design

> 작성일: 2026-02-15
> 상태: 승인됨

## 1. 프로덕트 비전

**핵심 가치**: "자식들이 이 앱만 깔아주면 안심하고 지낼 수 있도록"

시니어 챗(Seniar Chat)은 디지털 소외 계층인 시니어를 위한 AI 동반자 앱이다.
따뜻한 손자/손녀 같은 AI가 어르신의 모든 궁금증과 문제를 해결해주는 만능 안심 도우미.

### 투 사이드 제품

| 역할 | 대상 | 핵심 가치 |
|------|------|-----------|
| 사용자 | 60세 이상 시니어 | 무엇이든 물어볼 수 있는 따뜻한 AI 동반자 |
| 구매자 | 자녀/손자녀 | 부모님에게 깔아드리면 안심할 수 있는 앱 |

### 접근 경로
- 영상 콘텐츠 광고 제작 -> 앱스토어 설치 유도
- 타겟: 자녀 세대 (30-50대)가 부모님께 깔아드리는 시나리오

### 수익 모델
- Freemium 구독제
- 기본 무료: 일일 대화 횟수 제한
- 프리미엄: 무제한 대화, 음성, 전문 상담 연계 등

## 2. 시스템 아키텍처

```
[어르신 앱]                [자녀 앱 (Phase 2)]
React Native/Expo          React Native/Expo
     │                          │
     └──────────┬───────────────┘
                │ REST + SSE (WebSocket)
     ┌──────────┴──────────────────┐
     │   Spring Boot API Server    │
     │                             │
     │  Chat / Auth / Billing      │
     │  AI Orchestrator (Spring AI)│
     └──────────┬──────────────────┘
                │
     ┌──────────┼──────────────┐
     │          │              │
 Claude API  PostgreSQL     Redis
 (Anthropic)  (대화이력)    (세션캐시)
```

### 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 모바일 | React Native / Expo | 크로스플랫폼, 앱스토어 배포 |
| 백엔드 | Spring Boot (Java/Kotlin) | 엔터프라이즈급, 검증된 생태계 |
| AI | Spring AI + Anthropic Claude | 서버사이드 프롬프트 관리, API 키 보호 |
| DB | PostgreSQL | 대화 이력, 사용자, 구독 |
| 캐시 | Redis | 세션, 대화 컨텍스트 임시 저장 |
| STT | Whisper API (or Google Speech) | 한국어 음성 인식 |
| TTS | OpenAI TTS (or Clova) | 자연스러운 한국어 음성 합성 |

## 3. MVP 기능 상세 (Phase 1)

### 3.1 AI 대화

| 항목 | 상세 |
|------|------|
| AI 페르소나 | "따뜻한 손자/손녀" - 이름: 다솜이 (가칭) |
| 대화 범위 | 제한 없음 (건강, 생활, 디지털, 감정, 뉴스 등) |
| 말투 | 친근한 존댓말 ("할머니, 그건 이렇게 하시면 돼요~") |
| 응답 길이 | 짧고 명확 (3-5문장), 필요시 단계별 안내 |
| 대화 저장 | 서버 저장, 이전 대화 이어서 가능 |
| 안전장치 | 의료/법률 전문 분야는 "전문가 상담 권유" 필수 포함 |

**프로액티브 AI (핵심 차별점):**

다솜이는 수동적으로 질문을 기다리지 않는다. **먼저 유용한 정보를 알려주고, 먼저 질문한다.**
어르신들은 "뭘 물어봐야 하는지" 자체를 모르는 경우가 많기 때문.

첫 대화 플로우:
1. "안녕하세요, 다솜이예요! 어르신께 딱 맞는 정보를 알려드리려면, 어디 동네에 살고 계신지 알려주시겠어요?"
2. 동네 확인 후 -> 지역 맞춤 정보 제공 시작
3. "역삼동 주민센터에서 이번 달에 무료 건강검진을 해준대요. 관심 있으세요?"

프로액티브 정보 카테고리:
- **동네 공공서비스**: 주민센터/동사무소 프로그램, 시니어센터 이용 방법
- **복지 혜택**: 기초연금, 교통비 지원, 무료 건강검진, 냉난방비 지원
- **생활 안전**: 보이스피싱 주의보, 계절별 건강 관리, 낙상 예방
- **디지털 안내**: 카톡 사용법, 앱 설치 방법, 키오스크 사용법
- **문화/여가**: 동네 경로당, 문화센터 프로그램, 무료 공연/전시

지역 정보 활용:
- 온보딩에서 시/구/동 단위 수집
- 사용자 프로필에 저장 -> 시스템 프롬프트에 지역 컨텍스트 포함
- 지역별 주민센터, 시니어센터, 보건소 정보 연동 (Phase 2에서 API 연동)
- MVP에서는 Claude에게 지역 정보 기반 응답 유도

**핵심 사용 시나리오:**
- 다솜이가 먼저: "할머니, 강남구에서 이번 달에 무료 독감 예방접종 해준대요!"
- "이 약 먹어도 되나?" -> 일반 정보 + 반드시 의사 상담 권유
- "보이스피싱인 것 같은데..." -> 판단 도움 + 112 안내
- "손자한테 카톡 보내고 싶은데" -> 단계별 스크린샷 안내
- "오늘 무릎이 아프다" -> 공감 + 동네 병원/약국 안내
- "심심하다, 아무도 없다" -> 따뜻한 대화 + 동네 경로당/센터 안내

### 3.2 음성 인터페이스

| 항목 | 상세 |
|------|------|
| 음성 입력 (STT) | 큰 마이크 버튼 -> 말하면 텍스트 변환 -> 자동 전송 |
| 음성 출력 (TTS) | AI 응답 자동 읽어줌 (ON/OFF 토글) |
| 기술 | Whisper API (STT) + OpenAI TTS or Clova (TTS) |
| 한국어 | 한국어 최적화 필수 (사투리 인식은 Phase 2) |

### 3.3 인증 / 온보딩

| 항목 | 상세 |
|------|------|
| 가입 | 전화번호 인증 (시니어에게 가장 쉬운 방식) |
| 온보딩 | 4단계: 이름 -> 동네(시/구/동) -> 글씨 크기 선택 -> 음성 테스트 |
| 동네 수집 | 맞춤형 공공서비스/복지 정보 제공을 위한 핵심 데이터 |
| 자녀 연동 | 자녀가 대신 가입 후 어르신 폰에 로그인 가능 |

온보딩 플로우:
1. "어르신, 이름이 어떻게 되세요?" (호칭 수집)
2. "어디 동네에 살고 계세요?" (시/구/동 선택 or 직접 입력)
3. "글씨 크기를 골라주세요" (보통/크게/아주크게 - 실시간 미리보기)
4. "다솜이 목소리 들어보실래요?" (TTS 테스트 + 속도 조절)

### 3.4 시니어 UX

| 항목 | 상세 |
|------|------|
| 화면 | 채팅 화면 1개만 (복잡한 네비게이션 없음) |
| 글씨 | 기본 20px, "크게/보통/아주크게" 선택 |
| 버튼 | 56px 이상, 아이콘 + 텍스트 항상 병행 |
| 색상 | 따뜻한 톤, 대비 4.5:1 이상 |
| 피드백 | 모든 액션에 명확한 시각적/청각적 피드백 |

## 4. Phase 2 (출시 후)

- 자녀 앱 (활동 리포트, 알림)
- 구독 결제 (인앱 결제)
- 사투리 인식 강화
- 이미지 인식 ("이 약 뭐야?" 사진 찍기)
- 일정/약 복용 리마인더
- 긴급 SOS (자녀에게 알림)
